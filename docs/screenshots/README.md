# Screenshots

Place real PNG screenshots in this directory using the exact filenames below. Capture the local demo app with seeded demo data, not a real production server.

| File | What should be in the screenshot |
| --- | --- |
| `login.png` | Login screen showing the DaemonDeck sign-in form with empty or demo-safe fields. |
| `overview.png` | Signed-in dashboard overview showing account details, server health cards, host details, and session controls. |
| `metrics.png` | Metrics tab showing live status, refresh interval selector, trend chart, CPU/memory/disk cards, and at least one metric table or meter. |
| `processes.png` | Processes tab showing search/sort controls, process table, managed restart actions if available, and the process details modal opened. |
| `admin.png` | Admin tab signed in as `demo`, showing user role controls, alert rules, and system action buttons. |

Use a desktop viewport around 1440 x 1000 for README-friendly screenshots. Re-capture these files after UI changes that affect layout, navigation, or visible data.
